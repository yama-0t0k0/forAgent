主要機能とUI仕様

ミニヒートマップ抽出ロジック

全90タイルのスキルデータを「3×4」のグリッドに要約して表示。 スライディングウィンドウ方式により、最もスキルの密度が高い領域を自動抽出します。

抽出イメージ (High-Density)
Algorithm: Sliding Window
  • 表示サイズ: 縦3 × 横4 (縮小係数 0.7適用)
  • 判定対象: スキル経験のみ (志向は除外)
  • ツールチップ: タップ時に スキル名 + レベル を表示

詳細モーダル / 画面仕様

UserDetailModal (個人詳細)

画面占有率80%。ヒーローヘッダー、スキルバッジ、90タイルのフルヒートマップを統合表示。

CompanyDetailScreen (法人詳細)

Corporate AppのUIを再利用。データ構造の差異（フラット⇔ネスト）を吸収するマッピング処理を実装。

JobDetailModal (求人詳細)

ポジション名、企業名、スキル要件を可視化。ピンク（必須）、青・橙（歓迎）のカラーバッジを採用。

データ整合性と安全化方針

厳格な文字列化

Text要素には必ず String(v) を適用。オブジェクト混入による「Objects are not valid as a React child」エラーを完全に防止します。

ネストの吸収

基本情報.姓 が無い場合は name を参照するなど、複数アプリ間でのFirestoreスキーマ差異を考慮したフォールバック。

住所の整形

formatAddress ユーティリティにより、郵便番号から建物名までを安全に結合し、未設定時は一貫して "-" を表示。

将来的な改善・拡張案

正確な時系列分析 Pending

現在はモック配布。Firestoreに createdAt フィールドが追加され次第、リアルタイムな月別成長率グラフへ移行。
