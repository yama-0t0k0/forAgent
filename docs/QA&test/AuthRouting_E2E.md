# E2Eテスト計画・詳細設計: ロールベース・ルーティング検証

本ドキュメントは、LPアプリを通じた認証と、ロールに基づく専用アプリ（ローカルポート）への自律的リダイレクト機構を検証するためのテスト詳細設計です。

## 1. テスト概要
`lp_app` (Web) 上でログインした際、認証されたアカウントの Custom Claims の `role` に応じて適切なポートへフェデレーション・リダイレクトされることを保証する。

## 2. 要件定義と期待結果
- [ ] Adminロールでのログイン後、`localhost:8081` への遷移が試行されること。
- [ ] Corporateロールでのログイン後、`localhost:8083` への遷移が試行されること。
- [ ] Individualロールでのログイン後、`localhost:8082` への遷移が試行されること。

## 3. テストユーザー情報
このテストは専用のE2E自動セットアップ済みアカウントで行います（パスワード共通: `E2ePassword2026!`）。

| ロール | UID | ログインEmail | リダイレクト先 (ローカル) |
| :--- | :--- | :--- | :--- |
| **Admin** | `A999` | e2e-test-A999@lat-inc.com | `http://localhost:8081` |
| **Corporate** | `B00001` | e2e-test-B00001@lat-inc.com | `http://localhost:8083` |
| **Individual**| `C000000000001` | e2e-test-C000000000001@lat-inc.com | `http://localhost:8082` |

## 4. 詳細設計
### テストスタック
- **ツール**: Agent Browser (または Playwright)
- **環境**: ローカル開発サーバー (`localhost:8087` 等)
- **対象アプリ**: `lp_app` (Auth Entrypoint)

### 検証ロジック (Step-by-Step)
1. `http://localhost:8087` にアクセスする。
2. LP画面内の「LPアプリへ」などの導線（実装に依存）から、ログイン画面へ遷移する。
3. まず **Adminアカウント** のEmailとパスワードを入力し、ログインを実行する。
4. ログイン後、URLが `http://localhost:8081` に書き換わっている（あるいはDev Brokerがインターセプトする動きを見せる）ことを確認する。
5. （一度サインアウトするかシークレットモードを利用し）**Corporateアカウント** で同様にログインし、URLが `http://localhost:8083` に向かうことを確認する。
6. 同様に **Individualアカウント** でログインし、URLが `http://localhost:8082` に向かうことを確認する。

## 5. 備考
各ドメイン（ポート）でExpoアプリが実際に立ち上がっていなくても、URLのリダイレクト先が `808x` に切り替わった段階でLPアプリとしての責任・ルーティングテストとしては **PASS** となります。
