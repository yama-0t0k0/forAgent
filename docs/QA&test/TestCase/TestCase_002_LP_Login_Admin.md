# テスト設計書: TestCase 002

## 1. テスト概要
*   **テストID**: TC-002
*   **テスト名**: 管理者ログインおよびアプリケーション間リダイレクトの検証
*   **対象アプリ**: LPアプリ (`lp_app`) ➔ 管理画面アプリ (`admin_app`)
*   **検証目的**: 
    1.  Firebase Authを用いた管理者ログインが正常に完了すること。
    2.  ユーザーのロール（admin）に基づき、適切なURL（ポート8081）へリダイレクトされること。
    3.  リダイレクト先のアプリ（Admin App）で認証状態が維持され、ダッシュボードが表示されること。

## 2. 前提条件
*   **実行環境**: Web版 (Playwright)
*   **テスト用データ**:
    *   `A999` アカウントが Firestore/Auth エミュレータに存在すること。
    *   ID: `e2e-test-A999@lat-inc.com`
    *   Password: `E2ePassword2026!`
    *   Role: `admin`
*   **起動プロセス**:
    *   `lp_app` がポート 8087 で起動していること。
    *   `admin_app` がポート 8081 で起動していること。

## 3. テスト手順 (Web E2E)

| 順序 | 操作内容 | 期待結果 |
| :--- | :--- | :--- |
| 1 | `http://localhost:8087` を開く | LPトップ画面が表示される |
| 2 | 「管理者ログイン」ボタンをクリック | ログイン画面（`/login`）へ遷移する |
| 3 | メールアドレスとパスワードを入力 | 入力した値が正しく表示されている |
| 4 | 「ログイン」ボタンをクリック | 認証処理が開始される（ローディング表示） |
| 5 | URLの変化を待機 | 自動的に `http://localhost:8081` へリダイレクトされる |
| 6 | 画面要素の確認 | Admin Appの「管理ダッシュボード」が表示される |

## 4. エビデンス取得ポイント
*   **Login_Input**: 認証情報を入力した直後の状態。
*   **Redirect_Complete**: リダイレクト後のAdmin Appダッシュボード表示状態。
*   **Execution_Log**: Playwrightによるアサーション結果およびコンソールログ。

## 5. 自動化のポイント
*   **マルチアプリ起動**: `scripts/test_web_e2e.sh` を拡張し、テスト前に `lp_app` と `admin_app` の両方をバックグラウンドで立ち上げる。
*   **クロスドメイン待機**: Playwrightの `waitForURL` を使用して、ポート 8087 から 8081 への遷移を確実にキャッチする。
