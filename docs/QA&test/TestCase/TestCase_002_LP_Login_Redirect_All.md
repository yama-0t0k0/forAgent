# テスト設計書: TestCase 002

## 1. テスト概要
*   **テストID**: TC-002
*   **テスト名**: 全ロール対象：ログインおよびアプリケーション間リダイレクトの検証
*   **対象アプリ**: LPアプリ (`lp_app`) ➔ 各ロールのアプリ (`admin_app`, `individual_user_app`, `corporate_user_app`)
*   **検証目的**: 
    1.  管理者、個人、法人の各アカウントで正常にログインできること。
    2.  各ロールに応じた正しいポート（8081, 8082, 8083）へリダイレクトされること。
    3.  リダイレクト先の各アプリで認証状態が認識され、それぞれのダッシュボードが表示されること。

## 2. 前提条件
*   **実行環境**: Web版 (Playwright, --workers=1)
*   **テスト用データ**:
    *   Firestore/Auth エミュレータに以下のアカウントが存在すること。
    
| ロール | ユーザーID | メールアドレス | パスワード | 遷移先ポート |
| :--- | :--- | :--- | :--- | :--- |
| Admin | `A999` | `e2e-test-A999@lat-inc.com` | `E2ePassword2026!` | 8081 |
| Individual | `C000000000001` | `e2e-test-C000000000001@lat-inc.com` | `E2ePassword2026!` | 8082 |
| Corporate | `B00001` | `e2e-test-B00001@lat-inc.com` | `E2ePassword2026!` | 8083 |

*   **起動プロセス**:
    *   テスト実行時に `lp_app` および遷移対象のアプリが起動していること。

## 3. テスト手順 (各ロール共通)

| 順序 | 操作内容 | 期待結果 |
| :--- | :--- | :--- |
| 1 | `http://localhost:8087` を開く | LPトップ画面が表示される |
| 2 | ヘッダーの「ログイン」ボタンをクリック | ログイン手段選択画面が表示される |
| 3 | 「Password でのログインはこちら」をクリック | ログイン画面（`/login`）へ遷移する |
| 4 | 各ロールのメールアドレスとパスワードを入力してログイン | 認証処理が正常に完了する |
| 5 | URLの変化を待機 | ロールに対応したポートへリダイレクトされる |
| 6 | 画面要素の確認 | 遷移先アプリの「ダッシュボード」または「マイページ」が表示される |

## 4. エビデンス取得ポイント
*   **[Role]_Login_Input**: 各ロールで認証情報を入力した状態。
*   **[Role]_Redirect_Success**: リダイレクト後の各アプリ画面。
*   **Execution_Log**: 全パターンの実行結果ログ。

## 5. 自動化・実装のポイント
*   **マルチアプリ制御**: `lp_app` に加え、検証対象の全アプリをバックグラウンド起動する仕組み。
*   **テストの並列化/順次化**: Playwright で 3 パターンのテストを独立して実行。
*   **セッション管理**: Originを跨ぐため、各テストごとにクリーンなブラウザコンテキストを使用。
