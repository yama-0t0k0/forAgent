# テスト設計書: テストケース1 (LPアプリ Golden Path)

## 1. テスト概要
本テストケースは、プロジェクトのエントリーポイントとなる **LPアプリ (Landing Page)** の可用性と主要UIの描画を担保するための「王道ルート（Golden Path）」検証です。ローカル開発環境での検証サイクルを最速で回すため、**WebブラウザでのE2Eテストに特化**し、ユーザーがアプリを開いた際に正常にコンテンツが表示されることを証明します。
*(※当面はWeb版による高速なフィードバックループを構築し、これが安定稼働した後にモバイル環境等への横展開を検討します)*

## 2. テスト対象とスコープ
* **対象アプリ**: `apps/lp_app`
* **検証レイヤー**: Playwright を用いた Web レイヤー (PCブラウザ / Chromium ヘッドレス環境)
* **スコープ**: アプリの起動（ローカルサーバーアクセス）から、主要なCTA（Call to Action）要素である「登録」関連文言が画面上にレンダリングされるまで。

## 3. テスト要件（Acceptance Criteria）
1. ローカルWebサーバー (`http://localhost:8087` 等) にアクセスした際、アプリが正常に起動すること。
2. 画面上に `.*登録.*` を含むテキスト要素がレンダリングされ、視認可能 (Visible) になること。
3. テスト実行後、結果に関わらず最終画面のスクリーンショットと実行ログが規定の命名規則で保存されること。

## 4. 詳細設計（テストステップ）

### Web 版 (Playwright) 実行フロー
* **実行スクリプト**: `scripts/test_web_e2e.sh lp_app`
* **テスト定義**: `tests/e2e_web/lp_app.spec.ts`
* **ステップ詳細**:
  1. バックグラウンドで `npx expo start --web --port 8087` を起動し、Webサーバーの立ち上がりを curl ポーリングで待機。
  2. `page.goto('/')` : ローカルサーバーのルートURLへアクセス。
  3. アプリケーションのマウント先である `#root` 要素の DOM アタッチを待機。
  4. `page.getByText(/.*登録.*/).first()` 要素が画面に出現するまで最大15秒待機（アサーション）。
  5. 処理終了後、フルページのスクリーンショットを撮影。
  6. Playwrightの標準出力をログファイルとして出力し、バックグラウンドのExpoサーバーを安全に停止（クリーンアップ）。

## 5. エビデンス（証拠保存）規定
テストの安定性と監査可能性を担保するため、本ケースでは以下の厳格なエビデンス保存ルールを適用します。

* **保存先ディレクトリ**: `tests/evidence/`
* **命名規則**: `[アプリ名]_web_[YYYYMMDD_HHmmss]_[証拠種別].[拡張子]`
  * 例 (ログ): `lp_app_web_20260422_113431_playwright.log`
  * 例 (画像): `lp_app_web_20260422_113431_screen_final.png`
* **必須要件**: タイムスタンプは必ず**ローカル時間**の `YYYYMMDD_HHmmss` フォーマットを使用すること（UTCや時刻省略は不可）。`test_web_e2e.sh` 側で生成したタイムスタンプ環境変数を用いて、ログファイルとスクリーンショットの時間をミリ秒単位で完全に同期させることを徹底します。
