E2E テスト計画・詳細設計

プロフィール編集フローの堅牢性を Maestro で担保するための戦略と、証拠記録プロトコル。

テスト概要

`individual_user_app` のプロフィール編集画面への遷移と、統合された `PureRegistrationScreen` の内容（タブ構造およびメタデータ主導のUIレンダリング）が正常であることを検証します。

  • フッターから編集画面への遷移検証
  • 4つの主要タブの存在と表示検証
  • 実行エビデンスの自動保存プロトコル

テスト手法 (Test Stack)

Automation Tool

Maestro (E2Eテスティングフレームワーク) を採用し、YAML 形式で直感的なシナリオを記述。

Environment

iPhone 13 / iOS 18.4 シミュレータ上で、Expo 開発サーバーを経由して実行。

エビデンス管理 (Evidence Management)

成功時のみ動画をアーカイブ

`maestro record --local` を使用し、`tests/utils/record_success.sh` で成功判定時のみ永続化。

# 成功時の保存パス
