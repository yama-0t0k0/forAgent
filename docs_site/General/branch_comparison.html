ブランチ比較レポート

main vs yama。単一アプリからスケーラブルなモノレポ構造への刷新ログ。

レポート概要

main ブランチと yama ブランチを比較した結果、プロジェクト構造が**単一のExpoプロジェクトからモノレポ（Monorepo）構造へ大幅に刷新**されていることがわかりました。

トップレベルディレクトリの比較

ディレクトリ / ファイル main yama 備考
apps/ ❌ ✅ 各アプリ（admin, individual, corporate等）
shared/ ❌ ✅ バックエンド、フロントエンド、共通ロジック
infrastructure/ ❌ ✅ Firebase設定やTerraform等のインフラ関連
scripts/ ❌ ✅ 自動化スクリプト等
tests/ ❌ ✅ 統合テスト・Firestore Rulesテスト等
melos.yaml ❌ ✅ Melos (Dartモノレポツール) 設定

ソースコード・マッピング

From: main / (ルート直下のExpoコード)
To: yama apps/individual_user_app/expo_frontend/
From: main (templates) /assets/json/templates/...
To: yama apps/*/expo_frontend/assets/json/

yama ブランチでの進化

  • 複数アプリの並行開発: admin, individual, corporate, jd など役割別のサービスを独立して管理。
  • 共通部品の抽出 (shared/): UI、API通信、データモデルをアプリ間で共有可能に。
  • インフラのコード化: Firebase エミュレータやセキュリティルールが各環境ごとに整備。
