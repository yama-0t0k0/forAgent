主要な教訓 (Core Learning)

Firestore セキュルール不備事案

Date: 2026-02-14 / Admin App / High Priority
Incident Summary

開発スピードを優先し、テスト用ユーザーの `isAdmin` フラグを手動で Firestore に追加した際、セキュリティルールが `get()` による DB 参照を行っていなかったため、認可エラーが発生した。

Solution & Prevention
  • ・Custom Claims を主体とし、DB フォールバックを持つ Hybrid 認可へ移行。
  • ・デプロイ前に `firebase emulators:exec` を通じたルール検証を必須化。

テスト通過の目的化による実機不具合

Date: 2026-02-07 / Admin App / High Priority

E2EテストをパスさせるためにAPI通信をモックで代替した結果、本番環境でのURL設定ミスが隠蔽され、実機で「つながり候補一覧」が表示されない重大な不具合を招いた。

Core Learning: テスト通過は手段であり、ゴールは「実環境での動作」である。モック利用は最小限に留め、コード修正後は必ず `start_expo.sh` での手動確認を行うこと。

Milestone 更新指示の誤認

Date: 2026-03-04 / Process / Low Priority

「Milestoneを更新せよ」という指示をドキュメントの更新のみと解釈し、GitHub上のIssue Closeを失念。進捗率と実態の乖離を招いた。

Action: 「Milestone更新」は `gh issue close` を含む一連の操作と定義し、作業完了の定義をGitHub UI上の状態まで広げる。
