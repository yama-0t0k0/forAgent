クラスベース設計

Firestore のデータ I/O は生オブジェクト (POJO) ではなく、必ずモデルクラス (`User`, `Company`, etc.) を通じて行います。 

class User {
  constructor(data) {
    this.id = data.id;
    this.name = data.name || "Unknown";
  }

  static fromFirestore(doc) {
    return new User({ id: doc.id, ...doc.data() });
  }
}

バレルファイル (Barrel Files)

インポートを整理し AI エージェントの検索性を高めるため、ディレクトリごとに `index.js` を配置し、一括エクスポートを行います。

export * from './GenericRegistrationScreen';
export * from './PureRegistrationScreen';

モバイルファースト UI 実装ルール

  • SafeArea 遵守

    ノッチやホームインジケータに干渉しないよう、`SafeAreaView` または共通の `ScreenContainer` を必須で使用します。

  • Tailwind Native

    スタイル定義はインラインや StyleSheet ではなく、可能な限り Tailwind ユーティリティを使用します。
