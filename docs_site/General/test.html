テスト方針

「実環境での検証」を最優先とし、UI 挙動と Firestore I/O の整合性を100%保証するためのクオリティガイドライン。

テストの三原則

1

Real Data & Real Auth

可能な限りモックを避け、実際の Firebase Auth と Firestore（テスター専用環境）を使用した実機レベルの検証を行います。

2

E2E-First

単体テスト以上に、ユーザーのゴールが達成可能かを確認する E2E テストの網羅率を重視します。

3

I/O Integrity

画面が「保存完了」を表示するだけでなく、実際に Firestore 上に正しいスキーマで書き込まれたかをアサーションします。

テストカテゴリ

E2E Tests (Main Focus)

`scripts/run_e2e.sh` を使用。主要なカスタマージャーニー（登録、検索、承認）を自動走査します。

Security Verification

Firestore Security Rules の整合性を `jest` で検証。不正アクセスが正しく拒絶されるかをチェックします。
