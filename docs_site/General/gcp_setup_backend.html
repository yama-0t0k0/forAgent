GCP Setup Guide for Backend CI/CD

Apps/Backend (Cloud Run) の自動デプロイを実現するための環境構築手順

📦 1. Artifact Registry の準備

バックエンドの Docker イメージ（AOTバイナリ内包）を格納するリポジトリを作成します。

gcloud artifacts repositories create backend-repo \
    --repository-format=docker \
    --location=asia-northeast1 \
    --description="Dart Backend Server Repository"

☁️ 2. Cloud Run API の有効化

自動チェック手順

  1. 1. Google Cloud Console で flutter-frontend-21d0a プロジェクトを選択。
  2. 2. 「API とサービス」 > 「有効な API とサービス」へ移動。
  3. 3. "Cloud Run Admin API" が一覧にあるか確認。
  4. 4. 無い場合は「API とサービスの有効化」から検索して「有効にする」をクリック。

🔑 3. GitHub Secrets の設定

GitHub Actions が GCP にアクセスするための認証情報をリポジトリへ登録します。

Secret 名 役割
GCP_SA_KEY デプロイ用サービスアカウントの JSON キー。

必須ロール（権限）:

  • ・Artifact Registry 書き込み
  • ・Cloud Run 管理者
  • ・サービス アカウント ユーザー

🚀 4. デプロイの実行

main ブランチへの apps/backend/** または shared/** のプッシュにより、パイプラインが自動起動します。 初回デプロイでは backend-app という名称のサービスが asia-northeast1 リージョンに自動生成されます。
