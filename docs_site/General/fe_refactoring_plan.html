FE リファクタリング計画

エンジニア体験（DX）の向上、Sharedモジュール化の徹底、および Flutter への戦略的刷新プロセス。

戦略目標 (Strategic Goals)

DRY の徹底 (Don't Repeat Yourself)

重複する UI とロジックを `shared/common_frontend` へ 100% 集約し、アプリ間での一貫性を担保。各エンティティにはモデル (User, Job, etc.) を通じてアクセスを行う。

モバイルファースト & Dart指向

JS環境下においても Dart のような型安全・クラスベースの設計を採用。Flutter への移行を「焼き直し」ではなく「スムーズな昇華」へ。

DX 向上に向けた優先事項

  • 共通コンポーネントライブラリの拡充

    `GenericSearchBar.js` や `GenericDataList.js` などの、各アプリで汎用的に利用できるコンポーネントを Shared へ順次追加。

  • 起動スクリプトの統一

    `scripts/start_expo.sh` へのパラメータ集約により、全アプリの起動コマンドを統一。
