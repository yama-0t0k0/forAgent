Dart コーディング規約

Full-stack Dart アーキテクチャを支える、高度な型安全とリアクティブな設計思想。

強力な型安全 (Highly Type-Safe)

`dynamic` の使用は原則禁止し、全てのモデルクラスにおいて Null safety を徹底します。 

class UserModel {
  final String id;
  final String? name; // Nullable allowed with explicit handle

  UserModel({required this.id, this.name});

  factory UserModel.fromMap(Map<String, dynamic> map) {
    return UserModel(id: map['id'], name: map['name']);
  }
}

非同期処理 (Async/Await)

全ての I/O 操作には `Future` または `Stream` を使用し、例外処理を徹底します。

try {
  await firestoreService.updateProfile(data);
} catch (e) {
  logger.error("Profile update failed: $e");
}

Clean Architecture 志向

  • Domain Models 共有

    `domain_models` パッケージをフロントエンドとバックエンドで共有し、データの不整合をゼロにします。

  • Dependency Injection

    テスタビリティを高めるため、`get_it` を使用した依存性注入を採用します。
