認証・認可設計

Custom Claims を用いたハイブリッド認証と、パスキー完全対応のセキュア基盤。

基本方針

パフォーマンス、コスト、セキュリティのバランスを最適化するため、Custom Claimsを用いたハイブリッド認証を採用します。

爆速判定 (Zero Latency)

Firestoreへの都度アクセスなし。トークン解析のみで権限チェックを完了。

堅牢なパスキー対応

FaceID/指紋認証などの生体認証を主軸とし、セキュリティとUXを両立。
