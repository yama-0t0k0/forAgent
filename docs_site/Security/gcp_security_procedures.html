GCP セキュリティ強化手順

エンジニア登録アプリのプロダクション環境における IAM 最小権限の徹底、API キー制限、およびデータ移行の標準手順。

1. IAM 権限の確認と最小化

Least Privilege Principle

  • 1 オーナー (Owner) や 編集者 (Editor) ロールを特定。
  • 2 Cloud Datastore User 等、必要最小限のロールへ置き換えます。

2. API キーの制限 (Restrictions)

フロントエンドで使用する API キーの不正利用防止。

ウェブサイト制限
https://flutter-frontend-21d0a.web.app/*
https://flutter-frontend-21d0a.firebaseapp.com/*
http://localhost:19006/*
API 制限 (必須)

Identity Toolkit API, Cloud Firestore API, Cloud Storage for Firebase API, Firebase Installations API を選択。

3. データ構造の移行実行

① 個人情報の分離

Individual データを public_profile と private_info に分割。

node scripts/migration/migrate_individual.js

② RBAC 基盤の整備

users コレクションの作成と role/companyId の一括付与。

node scripts/migration/create_users_collection.js

検証手順 (Deployment & Verification)

セキュリティルールの適用後は、必ず以下の E2E テストを実行し、Admin/Individual/Corporate 各ロールのアクセス権限不備がないことを検証してください。

E2E_MODE=REAL ./tests/run_e2e.sh
