バックエンドアプリ
apps/backend (Pure Dart)

プロジェクト全体のビジネスロジックを集約。全クライアントに統一的なAPIを提供する、Full-stack Dart アーキテクチャの心臓部。

システム構成

Platform

Google Cloud Run (Serverless)

Runtime

Dart Native (AOT Compiled)

Framework

Shelf / shelf_router

Container

Distroless / Scratch

データフロー (Target Architecture)

graph LR Client["Expo/Flutter Clients"] Backend["Cloud Run (Dart Server)"] Firestore[("Firestore")] Auth[("Firebase Auth")] Client -->|"HTTPS (Bearer Token)"| Backend Backend -->|"Verify Token"| Auth Backend -->|"Read/Write"| Firestore Backend -->|"Response (JSON)"| Client

Phase 2 以降、クライアントは Firestore を直接参照せず、この Backend API を経由してデータを取得・更新するアーキテクチャへ移行します。ビジネスロジックの一貫性と堅牢なセキュリティを担保します。

ディレクトリ構造

apps/backend/ ├── bin/ │ ├── server.dart // エントリーポイント (Shelfサーバー起動) │ └── verify_shared.dart // 共有ライブラリの参照検証 ├── lib/src/ │ ├── controllers/ // HTTPリクエストハンドリング │ ├── middlewares/ // 認証・ログ・エラー処理 │ └── services/ // ビジネスロジック層 ├── Dockerfile // Cloud Run 用マルチステージビルド └── pubspec.yaml // 依存関係 (domain_models 参照)
