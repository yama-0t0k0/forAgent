求人票管理
job_description (JD)

求人情報の定義、スキル要件、および志向ヒートマップの可視化。全アプリケーションから参照される独立したドメインアプリ。

システム構成

基本スペック

  • - Framework: Expo (React Native)
  • - Root Collection: `job_description`
  • - Model: `JobDescription.js` (@shared/src/core/models)

設計目的

ドメインモジュールとして独立させることで、Admin、Corporate、Individual のいずれからも共通の求人詳細 UI (`JobDescriptionContent.js`) を利用可能にします。

データフローとヒートマップ連携

graph LR A["Firestore (job_description/ID)"] B["JobDescriptionScreen (求人詳細)"] C["HeatmapCalculator (共有ロジック)"] D["HeatmapGrid (共有UI)"] E["DataContext (テンプレートフォールバック)"] A -->|onSnapshot| B E -->|initialData| B B -->|calculate data| C C -->|values 0..89| D

`HeatmapCalculator` を用いて、求人要件（必須・歓迎スキル）と企業志向を可視化。これにより、エンジニアとのマッチング度を直感的に把握可能。

データ構造定義 (JSON Example)

基本項目

{
  "求人基本項目": {
    "ポジション名": "フロントエンドエンジニア",
    "雇用形態": "正社員",
    "勤務地": "東京/リモート可",
    "年収レンジ": "600-900万円"
  }
}

スキル (必須/歓迎)

{
  "スキル経験": {
    "必須": {
      "TypeScript": {
        "実務経験あり": true
      }
    },
    "歓迎": {
      "Next.js": {
        "実務経験あり": true
      }
    }
  }
}
