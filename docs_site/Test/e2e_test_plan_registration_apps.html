E2E テスト計画書

登録・編集専用アプリの UI挙動 ＆ Firestore データ整合性検証

1. テスト目的と概要

本テストは、ユーザーが画面から入力した内容が、正しく Cloud Firestore の対応するコレクションに反映されるかを検証する「フルスタック自動テスト」です。 単なるUIの正常系確認にとどまらず、バックエンドデータとの不整合を未然に防ぐことを目的とします。

🤖 完全自動化

入力から検証まで自動実行

💾 データ整合性

Firestore I/Oの厳密な比較

📸 エビデンス保持

自動キャプチャとログ記録

2. テスト対象アプリケーション

対象アプリ 検証 URL Firestore Collection
エンジニア個人登録 engineer-app-site... public_profile
企業プロフィール登録 corporate-app-site... company
求人登録 admin-app-site... job_description

3. 実行・検証ステップ

STEP 1
検証前データの取得 (Pre-test Snapshot)
対象ドキュメントの現在の値を Firestore より取得し、初期状態として記録します。
STEP 2
入力・画面操作 (UI Automation)
各アプリのタブ（基本情報、志向、職歴等）を巡回し、全項目にテストデータを入力します。各遷移ごとにスクリーンショットを保存します。
STEP 3
保存実行 (Submit Action)
「保存」または「更新」ボタンをクリックし、正常完了通知を待ちます。
STEP 4
検証後データの取得と差分検証 (Post-test Diff)
再度 Firestore よりデータを取得し、入力データとの一致、および事前データからの変化を自動判定します。

4. テスト結果エビデンス要件

テスト完了後、以下のエビデンスを含む「テスト結果報告書」を自動生成します。

  • 入力データ一覧: 画面に入力した全パラメータ（JSON/Table形式）
  • UI スクリーンショット: 入力中の各タブ、および保存直後の完了画面
  • データ比較レポート (Diff):
    • 検証前 Firestore JSON
    • 検証後 Firestore JSON
    • 差分 (Modified Fields) のハイライト表示

5. テスト実行環境

Testing Framework: Playwright / Antigravity Agent Logic
Verification Method: Firebase Admin SDK (Direct Firestore Access)
Target Browser: Chromium (Headless/Headed)
