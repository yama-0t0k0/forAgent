Core Collections

Root Collection

users

認証済みユーザーの基本プロフィールと権限(RBAC)を管理する最重要コレクション。

  • roleadmin | corporate | individual
  • companyIdString (only for corporate)
  • dev_admin_grantString (emergency access)
🛡 Security Rule 本人は読み書き可能 (Role変更不可)。管理者はフルアクセス。
Root Collection

public_profile

エンジニアの公開プロフィール。他のユーザーや企業から検索・閲覧されるデータ。

  • nameString (Required)
  • photoURLURL String
  • headlineString
🛡 Security Rule 認証済みユーザーであれば全員閲覧可能。本人のみ編集可能。
Root Collection

private_info

エンジニアの個人情報(PII)。職務経歴書や連絡先など、機密性の高いデータを保持。

  • emailString
  • phoneNumberString
  • allowed_companiesArray<ID>
🛡 Security Rule 本人、管理者、または matched した企業のみ閲覧可能。
Root Collection

company

企業（法人）の基本情報。`corporate` コレクションとエイリアス関係。

  • nameString (Required)
  • industryString
  • websiteURL
🛡 Security Rule 全ユーザーが閲覧可能。所属企業のメンバーまたは管理者が編集可能。
Hierarchical

job_description

求人情報。企業ごとにサブコレクション `JD_Number` で階層化されている。

  • titleString (Required)
  • descriptionString (Required)
  • salaryRangeString (Required)
🛡 Security Rule 全ユーザーが閲覧可能。当該企業の担当者のみ書き込み可能。
State Machine

selection_progress

候補者と企業の選考プロセス。ステータス遷移やフェーズを管理。

  • id_individual_個人IDReference / UID
  • id_company_法人IDID
  • statusString
🛡 Security Rule 関係者（本人・企業担当者）および管理者のみがアクセス可能。
Analytics & Finance

FeeMgmtAndJobStatDB

紹介手数料や求人統計データ。収益管理とパフォーマンス計測に使用。

  • individual_IDUID
  • company_IDID
  • amountNumber (Positive)
🛡 Security Rule 管理者および関係する企業・個人が閲覧可能。

Security Architecture

Role-Based Access Control (RBAC)

Firestore Security Rules は、`users` コレクションの `role` フィールドまたは Firebase Auth の Custom Claims に基づいて動的に評価されます。

Admin

全コレクションへの書き込み・読み取り権限を持ちます。システムメンテナンスを想定。

Individual

自身のプロフィールの編集、公開プロフィールの閲覧、選考中の案件へのアクセスが可能です。

Corporate

自社の求人情報の編集、マッチングした候補者の非公開情報の閲覧が可能です。
