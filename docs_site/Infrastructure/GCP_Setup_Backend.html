GCP Setup for Backend

Cloud Run (Dart) と Artifact Registry を用いたプロダクション環境の CI/CD パイプライン構築。

Artifact Registry の構成

Dart バックエンドイメージを格納するための Docker リポジトリ作成。

gcloud artifacts repositories create backend-repo \
  --repository-format=docker \
  --location=asia-northeast1 \
  --description="Dart Backend Server Repository"

Cloud Run (Admin API) の有効化

既存の `matching-functions` が動作しているため、通常は有効化されていますが、以下の手順で再確認を推奨します。

  • ・コンソールから **APIs & Services > Enabled APIs & services** へ。
  • ・**"Cloud Run Admin API"** を検索し、Status が Enabled であることを確認。

GitHub Secrets の設定

デプロイを自動化するための認証キー (Service Account Key) の登録。

GCP_SA_KEY

Firebase 認可、Artifact Registry 書き込み、Cloud Run 設定変更の権限を持つ JSON キー。

Region

`asia-northeast1 (Tokyo)` を定数値として使用します。
